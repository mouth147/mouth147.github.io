Police Killings and Crime Rates

This was a semester long project that was built for a database class at Rutgers University. The topic given originally was gun control but it developed into something more relatable in today's time. Taking data from multiple sources I was able to create an interactive web application that allows the user to compare and contrast crimes in cities/states along with people that were killed by police officers. Users can make their own judgement to whether or not they believe crime rates provoke certain police killings, whther or not race has a big play in it, or whether or not the person that was killed was a victim due to a weapon or a crime they were committing.

The application was built using Python, Flask, HTML, CSS, Bootstrap, jQuery, and SQLite. The application uses Nginx, Gunicorn and is running on Amazon AWS. I take the data sources, separate it into relational tables and then store them into the SQLite database from a csv file. From there the user can query the database using the web application and produce graphs.

  • Date: December 2016
  • Live Application: policeandcrime.mikebotti.com

Edgewood Records

Flask Blog

The purpose of this project was to teach myself Python and Flask. I've heard many people talk about web applications at Hackathons and how much they loved Python so I wanted to give myself a project that I could benefit from. The first decision was between using Flask and Django. I chose Flask because it had a very low overhead and was extremely minimalistic compared to Django. If I needed a feature done, I would most likely have to implement my own version rather than just have it come pre-built using Django. I wanted the whole experience of a blog: user logins, image uploads, editing posts, creating new posts, an admin panel, etc. So over the course of a month I implented every feature I would want in a blogging website.

This project is currently hosted on Amazon AWS running on a Gunicorn application server and Nginx webserver. The project is built using Python, Flask, Bootstrap, HTML, CSS, SQLite, and Adobe Photoshop. The application fetches posts from the database and populates the pages with the data. Users can log in/log out, edit posts(if they have access), upload pictures, delete posts, view all posts, and write/edit draft posts.

  • Source Code: Github
  • Date: June 2016
  • Live Application: blog.mikebotti.com
